# Runbook Template

## Purpose

Describe what this runbook fixes or supports.

## Scope

State which systems this applies to, such as AD DS, Entra ID, Microsoft 365, VPN, or endpoint login.

## Preconditions

- User identity verified.
- Ticket created.
- Approval attached when access is being changed.
- Admin account has the minimum required permissions.

## Steps

1. Confirm the user, device, group, or app involved.
2. Check current state.
3. Make the approved change.
4. Validate with the user or with system output.
5. Document the result in the ticket.

## Rollback

Describe how to reverse the change if needed.

## Closure Note

Summarize the issue, action taken, and validation result.
