# Service Tickets

This folder holds realistic help desk tickets.

## Workflow

1. Start with a ticket in `backlog/`.
2. Complete the related lab.
3. Add internal notes and a user-facing resolution.
4. Copy or move the completed ticket into `resolved/`.

## Ticket Quality Checklist

- User identity verified when required.
- Business approval documented for access changes.
- Technical action is specific.
- Validation step is recorded.
- Closure note is clear and professional.
